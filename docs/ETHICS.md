# Ethical Guidelines & Legal Notice

## 📋 Code of Conduct

ZENXHAT adalah toolkit untuk **ethical reconnaissance** dan intelligence gathering. Pengguna HARUS mematuhi semua hukum yang berlaku.

### ✅ Penggunaan yang Diizinkan

1. **Penetration Testing Berlisensi**
   - Hanya dengan written authorization dari pemilik/administrator sistem
   - Dalam scope yang telah disepakati

2. **Security Research**
   - Penelitian vulnerability pada sistem milik sendiri
   - Publikasi hasil discovery dengan responsible disclosure

3. **Threat Intelligence**
   - Organisasi keamanan cyber menganalisis ancaman
   - Monitoring infrastruktur milik sendiri

4. **Pembelajaran Cybersecurity**
   - Lab/training environment pribadi
   - Kursus cybersecurity terakreditasi

5. **Bug Bounty Programs**
   - Dalam scope program yang ditawarkan
   - Mengikuti rules setiap platform

### ❌ Penggunaan yang DILARANG

1. **Unauthorized Access**
   - Akses sistem tanpa izin (HACKING)
   - Bypass security measures
   - Intrusion activities

2. **Privacy Violations**
   - Mengumpulkan data pribadi tanpa consent
   - Stalking atau harassment
   - Doxxing (publishing private information)

3. **Malicious Activities**
   - Scanning port untuk menemukan vulnerabilities untuk exploit
   - Creating botnets atau malware
   - Distribusi ransomware atau worm

4. **Illegal Intelligence**
   - Industrial espionage
   - Theft of trade secrets
   - Copyright infringement

5. **Data Theft**
   - Exfiltration data pribadi
   - Extortion atau blackmail
   - Sale of compromised data

## 🤝 Responsible Disclosure

Jika Anda menemukan vulnerability:

### 1. **Private Notification**
   - Email security contact (jangan public disclosure dulu)
   - Provide detailed proof-of-concept
   - Include timeline untuk fix

### 2. **Disclosure Timeline**
   - **Day 0**: Report vulnerability
   - **Day 30**: Follow-up if no response
   - **Day 90**: Public disclosure if no patch
   - **Day 120**: Publish details (jika masih unfixed)

### 3. **Good Faith**
   - Don't access/modify data beyond POC
   - Don't disrupt services
   - Don't share vulnerability sebelum fix

### Example Email
```
To: security@target.com
Subject: Responsible Disclosure - [Vulnerability Title]

Dear Security Team,

I discovered [vulnerability description] in your [system/service].

Details:
- Severity: [High/Medium/Low]
- Impact: [What attacker can do]
- Steps to reproduce: [Numbered steps]
- Proof-of-concept: [Link/Code]

I request 90 days before public disclosure.

Best regards,
[Your Name]
```

## 🔐 Security Best Practices

### When Using ZENXHAT

1. **Authorization First**
   - Dapatkan written permission
   - Klarifikasi scope
   - Set rules of engagement

2. **Documentation**
   - Log semua activities
   - Maintain audit trail
   - Record findings secara detail

3. **Data Protection**
   - Encrypt sensitive data
   - Secure findings reports
   - Destroy data setelah engagement

4. **Operational Security**
   - Use VPN untuk testing
   - Gunakan dedicated machine
   - Anonymize results
